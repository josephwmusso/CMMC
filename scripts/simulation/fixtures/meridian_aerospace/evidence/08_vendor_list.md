  The Vendor and Service Provider List is a 3-page Word document
  maintained by Jason Bell that catalogs Meridian Aerospace's third-party
  technology and service relationships. It was last updated in October
  2025.

  Page 1 lists Google Workspace (Business Starter plan) as the company's
  primary cloud platform for email, document storage via Google Drive, and
  collaboration tools. The entry notes the annual subscription cost, the
  admin account holder (jbell@meridian-aero.com), and the Google support
  tier (Standard support). No data processing agreement or security
  assessment of Google is referenced.

  Page 2 lists the SonicWall TZ370 firewall support contract. The entry
  includes the SonicWall partner/reseller name (a local IT shop), the
  support contract renewal date, and a note that firmware updates are
  applied "when the reseller visits." There is no defined SLA or patching
  cadence documented. The same page lists the Synology NAS (dev-nas)
  support contract through Synology's direct extended warranty program,
  noting it covers hardware replacement only.

  Page 3 contains a brief section titled "Other Vendors" listing the
  office internet service provider (local cable company), the
  printer/copier lease vendor, and the CNC machine tool vendor for
  maintenance of shop floor equipment. These entries contain basic contact
  information only.

  DELIBERATELY ABSENT: There is no managed service provider (MSP) listed
  for IT management, patching, or security monitoring — all IT is handled
  by Jason Bell personally. There is no security assessment, risk
  evaluation, or due-diligence review documented for any vendor. There are
  no flow-down clauses or DFARS 252.204-7012 requirements imposed on any
  vendor. There is no vendor access policy defining what systems vendors
  can access. There is no record of reviewing vendors' SOC 2 reports,
  certifications, or security postures. Microsoft Defender Antivirus
  (the built-in Windows protection) is used on endpoints but is not listed
  as a vendor relationship since it requires no separate contract.
